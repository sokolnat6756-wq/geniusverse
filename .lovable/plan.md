## Проблема

`/dashboard` — это страница для обычного пользователя: показывает тариф, предлагает «Выбрать тариф» и выбрать Гения. Админу это не нужно — у него своя панель (`/admin/geniuses`, `/admin/users`, `/admin/orders`).

## Решение

1. **Редирект админа с `/dashboard`** на `/admin/orders` (или `/admin/geniuses`). В `src/routes/_authenticated/dashboard.tsx` добавить проверку роли через серверную функцию `getIsAdmin` (уже существует) и, если админ, делать `navigate({ to: "/admin/orders" })`.

2. **Навбар для админа**: убрать ссылку «Кабинет» (`/dashboard`) у админа в `src/components/Navbar.tsx` — оставить только админские разделы. Логотип «Академия Гениев» по-прежнему ведёт на «/».

3. **После логина**: в `src/routes/login.tsx` после успешного входа проверить роль и редиректить админа в `/admin/orders` вместо `/dashboard`.

## Что не меняем

- Логику тарифов, RLS, страницы админки.
- Поведение для обычных пользователей.

## Файлы

- `src/routes/_authenticated/dashboard.tsx` — редирект для админа
- `src/components/Navbar.tsx` — скрыть «Кабинет» для админа
- `src/routes/login.tsx` — редирект после логина по роли
